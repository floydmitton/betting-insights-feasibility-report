# Out of Scope (Post-MVP)

The following are natural additions once the platform has paying subscribers and validated traction. None are included in the MVP build.

| Feature | Why It's Post-MVP | Estimated Effort |
| --- | --- | --- |
| **Native iOS / Android apps** | Responsive web covers mobile at launch; native apps require separate builds, app store approvals, and ongoing maintenance | 8–12 weeks per platform |
| **Live in-play odds** | Requires WebSocket real-time streaming infrastructure — fundamentally different from REST polling | 4–6 weeks + higher API costs |
| **Automated bet placement** | Requires direct sportsbook account API integrations, which most books don't offer publicly; massive legal/compliance implications | High complexity, legal review required |
| **Community features** | Discord bot integration, leaderboards, user profit tracking — valuable for retention but not core MVP | 3–5 weeks |
| **Advanced line shopping** | Sub-5-second odds refresh via OddsJam commercial API upgrade — requires WebSocket infrastructure | 2–3 weeks + significant API cost increase |
| **Affiliate programme management** | Platform for managing sportsbook affiliate partnerships and tracking commissions | 3–4 weeks |

## Recommended Post-MVP Roadmap

**Phase 2 (Month 2–3):** Community features + user profit tracking — drives retention and word-of-mouth

**Phase 3 (Month 3–5):** Native mobile apps — captures users who prefer app experience

**Phase 4 (Month 5+):** Live in-play odds + advanced line shopping — premium differentiator
